## План: tool calling для цен + typing preview + нормализация текста

### Часть 1: OpenRouter tool calling в `services/llm_service.py`
DeepSeek V4 Flash подтверждённо поддерживает `tools` и `tool_choice`.

**Изменения в `LLMService.chat()`:**
- Добавить параметр `price_service: PriceService | None = None`
- Добавить `tools` схему с одной функцией:
  ```python
  {
    "type": "function",
    "function": {
      "name": "get_floor_prices",
      "description": "Получить актуальные floor prices с маркетплейсов",
      "parameters": {
        "type": "object",
        "properties": {
          "market": {"type": "string", "enum": ["grapes", "mrkt", "portal"]}
        }
      }
    }
  }
  ```
- Цикл tool calling: если в ответе есть `tool_calls` → вызвать `price_service.get_floor_prices(market)` → добавить результат как `role: "tool"` message → повторный запрос к модели. Лимит итераций = 3 чтобы не зациклиться.
- Возвращать финальный `content` модели.

**Новый класс исключения:** `ToolCallError(LLMError)` — если price_service не передан, но модель вызвала инструмент.

### Часть 2: Message edit typing preview в `bot/handlers/chat.py`
Текущий SendMessageDraft не работает как "думаю..." статус. Заменю на паттерн edit-message:

1. **Сразу** отправляем `<i>думаю...</i>` через `bot.send_message`, запоминаем `msg_id`.
2. Если модель вызывала tool (price) — edit на `<i>получаю актуальные цены</i>` перед second LLM call.
3. После финального ответа — edit на реальный ответ бота.
4. `send_chat_action("typing")` каждые 4с параллельно (статус ~5с).

Для определения момента tool call — нужно поднять логику из LLMService в handler, ИЛИ сделать в LLMService callback `on_tool_call: Callable[[], Awaitable[None]] | None = None`. Второй вариант чище — handler передаёт callback который делает edit.

### Часть 4: Обновить `skills/instruction.md`
Добавить секцию с правилами форматирования и доступными HTML-тегами:
- Использовать **только HTML** (не Markdown) — `Message.answer(parse_mode=HTML)`.
- Доступные теги: `<b>`, `<i>`, `<u>`, `<s>`, `<code>`, `<pre>`, `<a href="...">`, `<blockquote>`.
- Когда показывать цены: вызывать tool `get_floor_prices` с параметром market (grapes/mrkt/portal).
- **Запрещено** выдумывать цены, rarities, коллекции которых нет в ответе tool.
- **Имя маркетплейса**: всегда "Portals" (не "Portal").
- После имени маркетплейса и в конце последнего предложения точку НЕ ставить.
