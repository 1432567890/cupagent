# План: исправление багов + Tonnel/XGift integration

## Контекст (что подтвердил пользователь)

- **Баг #1**: 17.3 на Порталс — это глобальный флор фона "Black" по ЛЮБОЙ коллекции (не Scared Cat). MRKT/GetGems правильно показывают ~1580, потому что их API возвращают collection-scoped floors. Portal API `/api/collections/filters?short_names=...` возвращает глобальные флоры фонов.
- **Баг #2**: числа `1581`, `1580` (целые) не обернуты в `<code>`, только `17.3` (десятичная). Регекс `_PRICE_NUM_RE` матчит только `N.NN`/`N,NN`.
- **Tonnel**: использовать `/filterStats` (6577 записей `Collection_Model: {floorPrice}`) для коллекций и моделей. Фоны пропускаем.
- **XGift**: JWT auth через Kurigram initData → POST `/auth/login`.

## 1. Баг #1 — Portal collection-scoped floors

**Файл**: `markets/portalsmp/client.py`

Заменить `fetch_collection_attributes` чтобы использовать listings endpoint `/api/nfts/search`:

- Построить карту `display_name → collection_id` (UUID) в `fetch_floor_prices` как побочный эффект (по аналогии с GetGems `_name_to_address`). GET `/api/collections` уже вызывается — нужно дополнительно извлечь поле `id` (UUID) вместе с `name` и `floor_price`.
- В `fetch_collection_attributes`:
  1. Резолвить `collection_name → collection_id` через закэшированную карту (с фолбэком на рефетч `/api/collections`).
  2. Для **фонов**: `GET /api/nfts/search?offset=0&limit=50&filter_by_backdrops={backdrop}&collection_ids={uuid}&status=listed&exclude_bundled=true&premarket_status=all` — НО это требует перечисления каждого фона. Альтернатива: один запрос БЕЗ `filter_by_backdrops`, затем агрегировать по полю `attributes[type=backdrop].value` и брать min(price) по каждому фону. Пейджингофсетный обход (по 50) пока `total_count` не исчерпан или не достигнем капа (3-5 страниц).
  3. Для **моделей**: аналогично — агрегировать min(price) по `attributes[type=model].value`.
  4. Вернуть `{"models": [...], "backdrops": [...]}` отсортированные по цене.

Это даёт **реальный** collection-scoped флор (как пользователь и ожидает: ~1580-2000 для Scared Cat + Black, а не 17.3).

**Почему это решает баг**: ответ Portal для "Black фон на Scared Cat" теперь будет ~2000 (как в примере URL из ответа пользователя: `"price": "2000"`), а не глобальный 17.3.

## 2. Баг #2 — context-aware price wrapping

**Файл**: `bot/handlers/_chat_core.py`

Текущий регекс (`_PRICE_NUM_RE`):
```python
r"(?<![<\w])(\d{1,6}[.,]\d{1,2})(?![\w])"
```
Матчит только десятичные. Нужно добавить контекстное правило для целых чисел после имени маркета или тире/двоеточия.

**Подход**: разделить на два правила —
1. **Десятичные** (как сейчас): `\d{1,6}[.,]\d{1,2}` — обернуть всегда.
2. **Целые в ценовом контексте**: целое число 2-7 цифр, перед которым стоит:
   - имя маркета (`грейпс`/`мркт`/`порталс`/`гетгемс`/`grapes`/`mrkt`/`portal`/`getgems`/`xgift`/`тоннел`/`tonnel` — любое слово из набора), опционально через пробел + `-`/`–`/`:`/пробел
   - ИЛИ `[-–:]` сразу перед числом (контекст "маркет - цена")
   
   Это ловит `мркт - 1581`, `порталс: 17.3` (уже десятичная), `гетгемс 1580`.
   
   **Защита от false positives**: НЕ оборачивать числа после `#` (номера подарков), НЕ оборачивать `Scared Cat 1581` (имя коллекции перед числом — gift-link wrapper уже обработал `#N`, но голое число после имени не трогаем), НЕ трогать 4-значные годы.

Реализация — заменить `_PRICE_NUM_RE.sub` в `_wrap_prices_in_code` на callback-функцию, которая матчит объединённым регексом:
```python
_PRICE_NUM_RE = re.compile(
    r"(?<![<\w])("
    r"\d{1,6}[.,]\d{1,2}"           # десятичная — всегда цена
    r"|"
    r"(?<=(?:грейпс|мркт|порталс|гетгемс|тоннел|grapes|mrkt|portal|getgems|tonnel|xgift)[\s:-]?)(\d{2,7})"  # целое после маркета
    r"|(?<=[-–:]\s)(\d{4,7})(?!\d)"  # целое 4-7 цифр после тире/двоеточия
    r")(?![\w])"
)
```
(финальный регекс уточню при имплементации — главное поведение описано выше)

## 4. XGift — новый клиент

**Файлы**: `markets/xgift/__init__.py` (дополнить), `markets/xgift/client.py` (новый)

**Auth flow**:
- Конструктор: `XGiftClient(init_data_provider, bot_username="xgift_bot", app_short_name="app")` (как MRKT/Portal).
- `authenticate()`:
  1. Получить initData через `_provider.get_init_data(bot_username, app_short_name)`.
  2. POST `/auth/login` с телом `{"dataType": "web", "data": "<initData>", "referredBy": null}`.
  3. Извлечь `access_token` (JWT), сохранить в `_jwt`.
  4. Self-healing как в MRKT (401 → invalidate initData → рефетч → retry).
- `_auth_headers()` → `{"Authorization": f"Bearer {self._jwt}"}`.

**Floor prices**:
- `fetch_floor_prices()`: GET `/collections/?page=1&limit=200` → `{gifts: [{giftName, giftNameFormatted, floorPrice, ...}]}`.
  - `floorPrice` — уже в TON (== GRAM 1:1), бывает int или string.
  - Построить карту `display_name → giftNameFormatted` (slug) как побочный эффект (для `fetch_collection_attributes`).
  - Вернуть список `FloorPrice`.

**Collection attributes**:
- `fetch_collection_attributes(collection_name)`:
  1. Резолвить `collection_name → slug` (giftNameFormatted, напр. "Scared Cat" → "ScaredCat").
  2. GET `/gifts/filters/{slug}` → `{giftModel: [{model, floorPriceTon}], giftBackdrop: [{backdrop, floorPriceTon}]}`.
  3. Извлечь модели и фоны с `floorPriceTon` (string → float TON).
  4. Вернуть `{"models": [...], "backdrops": [...]}`.
