# Skill: market-prices

## Назначение

Floor prices подарков с шести маркетплейсов: **Grapes**, **MRKT**, **Portals**, **GetGems**, **Tonnel**, **XGift**.

## Как работает

Вызови инструмент `get_floor_prices` с параметром `market`
(`grapes` / `mrkt` / `portal` / `getgems` / `tonnel` / `xgift`).

Возвращает JSON:

```json
{
  "market": "grapes",
  "total": 85,
  "prices": [
    {"name": "Lunar Snake", "price": 12.5},
    {"name": "Plush Pepe", "price": 6.39}
  ]
}
```

## Сопоставление коллекций по сленгу

Пользователь называет коллекции как угодно — сленгом, переводом,
описанием. Сопоставляй сами по списку имён из ответа инструмента:

- ищи по переводу: «лягушка» → ...Frog..., «змея» → ...Snake...
- по корню/подстроке: «плюшевый» → Plush...
- по смыслу: «звезда» → ...Star...

Несколько совпадений — покажи все варианты с ценами.
Не уверен — спроси уточнение.

## Правила

- НЕ выдумывай цены, rarities, коллекции — только из ответа инструмента
- Пустой список → скажи что данных нет
- Цена `null` или маркета нет → просто пропусти, НЕ пиши «null»/«None»
- Сравниваешь площадки → вызови для каждой
- Цены в **GRAM** (не TON — ребрендинг), максимум 2 знака после запятой

## Маркетплейсы

- **Grapes** — цена в GRAM, поле `floor`
- **MRKT** — цена в GRAM (преобразовано из nanoTON)
- **Portals** — цена в GRAM, поле `floor_price` (строка)
- **GetGems** — ончейн маркет, цена в TON (= GRAM 1:1)
- **Tonnel** — цена в TON (= GRAM 1:1), коллекционный флор считается как минимум из флоров моделей (`/filterStats`)
- **XGift** — цена в TON (= GRAM 1:1), поле `floorPrice`
